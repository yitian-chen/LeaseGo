# 前端改造指南

## 1. 服务地址变更

| 环境 | 旧地址 | 新地址 |
|------|--------|--------|
| API网关 | `http://localhost:8081` (原web-app端口) | `http://localhost:8083` |
| WebSocket聊天 | 无独立服务 | `ws://localhost:8082/app/chat` |

### 新架构说明

```
前端 → Gateway(8083) → web-app(8081) [业务接口 + 聊天HTTP接口]
                  └→ chat-service(8082) [仅WebSocket直连]
```

### 重要：接口调用规则

| 接口类型 | 调用方式 | 说明 |
|----------|----------|------|
| **所有 HTTP 接口** | 必须通过 Gateway (8083) 调用 | 包括聊天相关的 `/app/chat/conversations` 等接口 |
| **WebSocket 连接** | 直连 chat-service (8082) | 仅聊天功能使用 |

## 2. 登录接口变更

### 2.1 获取图形验证码

- **路径**：`GET /app/login/captcha`
- **返回**：
```json
{
  "code": 200,
  "data": {
    "image": "base64图片字符串",
    "key": "app:captcha:login:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }
}
```

### 2.2 登录请求

- **路径**：`POST /app/login`
- **Content-Type**：`application/json`

**请求体（短信验证码登录，strategy=1）**：
```json
{
  "strategy": 1,
  "phone": "138xxxx1234",
  "code": "123456"
}
```

**请求体（密码登录，strategy=2）**：
```json
{
  "strategy": 2,
  "phone": "138xxxx1234",
  "password": "xxxxxx",
  "captchaKey": "app:captcha:login:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "captchaCode": "abcd"
}
```

## 3. WebSocket 聊天接口

聊天 WebSocket **直连 chat-service**，不经过网关。

### 3.1 连接地址

```
ws://localhost:8082/app/chat?token=xxxxx
```

### 3.2 认证方式

通过 URL 参数 `token` 传递 JWT Token（与 HTTP 接口一致）

### 3.3 连接示例

```javascript
const token = '用户登录获取的JWT'; // 从登录接口获得
const ws = new WebSocket(`ws://localhost:8082/app/chat?token=${token}`);

ws.onopen = () => {
  console.log('WebSocket connected');
};

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  console.log('Received:', message);
};

ws.onerror = (error) => {
  console.error('WebSocket error:', error);
};

ws.onclose = () => {
  console.log('WebSocket disconnected');
  // 建议实现断线自动重连
};
```

### 3.4 消息格式

参考 `docs/联系房东功能接口文档.md`

## 4. 认证 Token 传递方式

### 4.1 HTTP 接口

所有通过 Gateway 调用的接口，必须在请求头中添加 `access-token`：

```javascript
fetch('http://localhost:8083/app/chat/conversations', {
  headers: {
    'access-token': '用户登录获取的JWT'
  }
})
```

### 4.2 WebSocket 连接

通过 URL 参数传递 Token：

```javascript
const ws = new WebSocket('ws://localhost:8082/app/chat?token=用户登录获取的JWT');
```

### 4.3 常见错误排查

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `未登录` / code=305 | 未携带 token 或 token 已过期 | 检查请求头是否包含 `access-token` |
| 频繁弹出未登录错误 | 轮询接口未携带 token | 确保所有请求都带有 `access-token` |
| WebSocket 连接失败 | token 过期 | 重新登录获取新 token |

## 7. 启动顺序

后端服务启动顺序：
1. Nacos (8848)
2. MySQL / Redis / MinIO
3. web-app (8081)
4. chat-service (8082)
5. gateway (8083)

前端开发服务器需确保在网关就绪后开始使用。
